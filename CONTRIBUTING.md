# Padrão de contribuição

## Branches
- main = produção
- develop = integração principal
- feature/* = novas funcionalidades
- fix/* = correções

## Commits
Utilizar padrão:

feat: nova funcionalidade
fix: correção
docs: documentação
refactor: reorganização

## Pull Requests
Todo código deve passar por revisão antes do merge.

## Regras
- Não trabalhar direto na main
- Atualizar o board do projeto
- Commits frequentes
- Código documentado minimamente
